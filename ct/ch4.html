Scout 영상 (위치확인 영상)
Scout View · Topogram · Surview · Pilot View · Scannogram

Scout 영상이란?

Scout View / Topogram / Surview / Pilot View / Scannogram

CT 검사의 시행 초기에 스캔범위(검사범위)와 방법 계획을 위해 시행하는 위치확인 영상. Frontal·Lateral projection 두 방향으로 촬영한다.

  • X선관(tube)과 검출기는 회전하지 않음 — 고정 상태
  • 테이블만 등속도로 이동
  • 관전류·관전압을 감소시켜 노출조건을 낮게 설정 → 환자 피폭선량 최소화

Scout 영상 기출 포인트

  1. Scout 영상의 다른 명칭: Scout view · Pilot view · Surview · Topogram · Scannogram (최소 3개 이상 암기)
  2. 튜브와 검출기는 회전하지 않고 테이블만 등속도로 이동한다.
  3. 관전류·관전압을 낮게(감소) 설정 → 피폭선량 최소화가 목적
  4. 촬영 방향: Frontal(정면) · Lateral(측면) 두 방향
  5. 목적: 스캔범위(FOV) 결정과 검사 방법 계획 수립

Sinogram (시노그램)

Sinogram · Raw Projection Data

다수의 각도 θ에서 얻은 투영 p(θ, t)를 2차원 평면에 나열한 데이터. 물체가 원점에서 벗어날수록 사인(sine) 곡선 형태로 나타난다.

  • x축 t (검출기 위치), y축 θ (회전 각도, 0 ~ π)
  • 영상 재구성(FBP 등)의 입력 데이터가 됨
  • 분류상 원 데이터(raw data)에 해당

Sinogram 기출 포인트

  1. Sinogram = 원 데이터(raw data) 영역 (= 감약 데이터의 총합 형태)
  2. 가로축: 위치(t), 세로축: 각도(θ, 0~π)
  3. Top range scanner에서는 프로젝션 수가 1,100 ~ 2,460개에 달함
  4. 프로젝션 수가 많을수록 공간해상능 ↑, 노이즈 ↓
  5. Ray sum = 한 방향에서의 ray들의 합

아날로그 → 디지털 변환과정
Analog-to-Digital Conversion (Sampling → Quantization → Coding)

A/D 변환 3단계

ADC Pipeline
1. 표본화
Sampling
2. 양자화
Quantization
3. 부호화
Coding

① 표본화

Sampling

아날로그→디지털 변환의 첫 단계. 컴퓨터 영상 재구성을 위한 데이터를 생성하는 과정.

  • 아날로그 파형을 디지털 파형으로 변화시키기 위해 표본을 취하는 것
  • 일정 시간 간격으로 아날로그 신호의 순간적인 값을 취함

② 양자화

Quantization

표본화로 쪼갠 연속적 값을 진폭의 크기에 따라 비연속적 대표값으로 변환하는 과정.

  • 예) 1.124323241541 → 대표값 1
  • 정수화 시 오차 발생 = 양자화 오차 (Quantization Error)

③ 부호화

Coding · Binary Encoding

표본화·양자화를 거친 디지털 정보를 0과 1의 이진수(binary)로 표현하는 과정.

  • 양자화된 정보는 전송 시 잡음에 매우 민감하므로 이진수로 부호화
  • 잡음 내성 확보 → 안정적인 디지털 전송 가능

A/D 변환 기출 포인트

  1. A/D 변환 순서 암기: 표본화(Sampling) → 양자화(Quantization) → 부호화(Coding)
  2. DAS를 거친 후의 데이터 명칭: 측정데이터(Measurement Data) — 5번 정답 기출
  3. 양자화 오차(Quantization Error) — 정수화 과정에서 필연적으로 발생
  4. DAS 구성: 검출기 → 증폭기 → 로그 증폭기 → ADC → Digital data → 디지털 전송 → 컴퓨터 메모리
  5. 부호화의 목적: 잡음에 대한 내성 확보

출처: 4장 CT영상의 형성, p.18~24

컨볼루션
Convolution · Convolution Kernel · Algorithm

Convolution (합성곱 · 여과처리)

Convolution · Mathematical Filtering

재구성되는 영상의 질을 결정하는 핵심 처리. 영상의 선예도(definition)와 노이즈에 많은 영향을 준다.

  • 3×3 convolution kernel을 입력영상의 각 픽셀에 대해 곱셈 후 합산
  • 공식: 새로운 P₅ = (K₁·P₁) + (K₂·P₂) + ... + (K₉·P₉)
  • Convolution 시행 전: 역투영 영상은 가장자리가 둥글고 흐림(blur) 상태
  • Convolution 시행 후: 가장자리가 선명하게 각진 형태로 재구성됨
New Pixel Value = Σᵢ₌₁⁹ Kᵢ × Pᵢ

Kᵢ · Convolution Kernel의 i번째 가중치 (3×3 matrix)

Pᵢ · 입력 영상의 i번째 픽셀 값

결과 · 중심 픽셀 P₅가 주변 8개 픽셀의 가중합으로 갱신됨 → 필터링 효과

  • B80 · Ultra sharp · High noise → 폐·골

Convolution 기출 포인트

  1. Convolution = 재구성되는 영상의 질을 결정하는 처리. 선예도와 노이즈를 좌우.
  2. 알고리즘 3종: Standard · Enhancement(Edge) · Smoothing
  3. Image detail ↑ → Image noise ↑ (trade-off 관계)
  4. 고해상 kernel (H70, B80): 골·폐 · 저해상 kernel (H30, B30): 뇌실질·연부조직
  5. 물 팬텀 스캔 후의 수학적 여과과정 전(좌) / 후(우) 영상 비교 → 기출 빈도 매우 높음
  6. Convolution kernel 연산: 3×3 matrix × pixel 값의 합

Back Projection (역투영법)

Back Projection · 역투영 알고리즘

영상 형성 과정의 가장 중요한 단계로, 우리가 눈으로 볼 수 있는 Image Data로 재구성하는 연산 알고리즘.

  • 각 방향에서 얻은 투영값(ray)을 거꾸로 픽셀면에 되돌려서 하나하나 합산하는 기법
  • 단순 BP의 한계점: 실제 좁은 projection 데이터가 넓게 퍼진 형태(blur)로 재구성됨 → 흐림(blurring) 발생

Histogram (히스토그램)

Histogram · Densitometry

영상의 일정 관심부위에서 CT값에 대한 픽셀의 빈도를 표시하는 영상의 통계함수.

  • x축 = 픽셀 농도(CT number), y축 = 빈도수(frequency)
  • 픽셀들의 위치에 관한 정보는 갖지 않음
  • 매트릭스 내 각 농도값에 대한 픽셀 수 통계를 그래프로 표시

후처리 기출 포인트

  1. Retrospective 재구성에서 SFOV는 변경 불가 — 기출 빈출 4번 정답
  2. Masking/Highlighting은 지방 성분 측정에 주로 사용
  3. Histogram은 위치 정보 없이 CT값의 빈도만 표시하는 통계함수
  4. 픽셀값 측정 시 ROI (관심영역) 설정 후 "Report Pixel" 기능 사용
  5. WW·WL 변경은 raw data 변경 없이 표시만 재조정 (비파괴적 후처리)

✓ 정답 · ⑤ 원 데이터 (raw data) Sinogram은 모든 projection을 각도 θ와 위치 t의 2차원 평면에 나열한 총 감약 데이터로, 영상 재구성의 입력이 되는 raw data의 대표적 표현 형태이다.
▎ PROBLEM 02 빈출 · BASIC
자료수집장치(DAS)를 거친 후의 데이터의 명칭은?
  • ① 원 데이터(raw data)
  • ② 영상 데이터(image data)
  • ③ 필터보정원데이터(filtered raw data)
  • ④ 전처리데이터(pre-processing data)
  • ⑤ 측정데이터(measurement data)
✓ 정답 · ⑤ 측정데이터 (measurement data) DAS 직후 단계의 데이터는 Measurement Data이며, 이후 전처리를 거쳐 Raw Data(=Scan Data)로 변환된다.
순서: Measurement → Raw → Filtered Raw → Image
▎ PROBLEM 03 계산 · INTERMEDIATE
유효 검출소자 수가 1,024개이고 초당 sampling 수가 2,000회인 CT 장치에서 1초 동안 수집되는 총 감약 데이터의 수는?
  • ① 512 × 10³
  • ② 1,024 × 10³
  • ③ 2,048 × 10³
  • ④ 4,096 × 10³
  • ⑤ 8,192 × 10³
✓ 정답 · ③ 2,048 × 10³ [STEP 1] 공식: 총 감약 데이터 = 유효 검출소자 수 × 초당 sampling 수
[STEP 2] 대입: 1,024 × 2,000 = 2,048,000
[STEP 3] 2,048,000 = 2,048 × 10³ → ③
▎ PROBLEM 04 개념 · BASIC
아날로그 신호를 디지털 신호로 변환하는 A/D 변환 과정의 순서로 옳은 것은?
  • ① 양자화 → 표본화 → 부호화
  • ② 표본화 → 양자화 → 부호화
  • ③ 부호화 → 양자화 → 표본화
  • ④ 표본화 → 부호화 → 양자화
  • ⑤ 양자화 → 부호화 → 표본화
✓ 정답 · ② 표본화 → 양자화 → 부호화 S·Q·C 순서로 암기 (Sampling → Quantization → Coding)

▎ PROBLEM 05 빈출 · APPLIED

  • ① 여과처리 후 영상
  • ② 여과처리 전 영상
  • ③ 선경화 보정 전 영상
  • ④ 선경화 보정 후 영상
  • ⑤ 나선식 스캔으로 얻은 영상
✓ 정답 · ② 여과처리 전 영상 수학적 여과(Convolution) 처리 이전의 back projection 결과는 단순 역투영의 한계로 인해 중앙이 높고 가장자리로 흐려지는 봉우리 형태를 보인다.

▎ PROBLEM 06 개념 · BASIC
다음 중 Scout 영상의 특징으로 옳지 않은 것은?
  • ① 일반적으로 Frontal과 Lateral 두 방향으로 촬영한다.
  • ② 튜브와 검출기가 고속으로 회전하며 데이터를 수집한다.
  • ③ 관전류·관전압을 감소시켜 노출조건을 낮게 설정한다.
  • ④ 스캔 범위(FOV) 설정을 위한 위치확인 영상이다.
  • ⑤ Topogram, Surview, Scannogram 등으로도 불린다.
✓ 정답 · ② 튜브와 검출기가 고속으로 회전하며 데이터를 수집한다. Scout 영상은 튜브와 검출기가 회전하지 않는다. 오직 테이블만 등속도로 이동하면서 일반 X선 사진과 유사한 투영 영상을 획득한다.

CT 영상 재구성 단계에서 데이터 명칭의 변화를 순서대로 바르게 나열한 것은?

  • ② Measurement → Raw → Filtered raw → Image

  • ④ Raw → Filtered raw → Measurement → Image

▎ PROBLEM 08 기출 · BASIC
(가) 원 Scan 영상에서 (나) Retrospective 재구성 영상으로 전환할 때 변경할 수 없는 파라미터는?
  • ① Matrix (매트릭스)
  • ② DFOV (Display FOV)
  • ③ Algorithm (재구성 커널)
  • ④ SFOV (Scan FOV)
  • ⑤ 좌표 (X, Y)
✓ 정답 · ④ SFOV SFOV는 스캔 시점에 물리적으로 확정되는 값이므로, 이미 획득된 raw data만으로는 사후에 변경할 수 없다. 반면 Matrix, DFOV, Algorithm, 좌표는 모두 retrospective하게 재구성 가능하다.
▎ PROBLEM 09 응용 · INTERMEDIATE
고해상도 폐 CT를 시행할 때 적절하게 선택될 수 있는 convolution kernel과 그에 따른 특성은?
  • ① H30 · 선예도 ↑ · 노이즈 ↓
  • ② B30 · 선예도 ↓ · 노이즈 ↑
  • ③ B80 · 선예도 ↑ · 노이즈 ↑
  • ④ B80 · 선예도 ↓ · 노이즈 ↓
  • ⑤ H70 · 선예도 ↓ · 노이즈 ↑
✓ 정답 · ③ B80 · 선예도 ↑ · 노이즈 ↑ B80은 Body 영역 중 가장 강한 edge-enhancement kernel로 폐야의 미세 구조(interstitium)를 선명히 보여준다. 대신 선예도와 노이즈는 trade-off 관계이므로 noise도 함께 증가한다.
(H-series = 두부용, B-series = 체부용)
▎ PROBLEM 10 개념 · APPLIED
CT 영상에서 히스토그램(Histogram)의 특성으로 옳지 않은 것은?
  • ① ROI 내 CT값에 대한 픽셀 빈도를 표시한다.
  • ② x축은 픽셀 농도(CT number), y축은 빈도수이다.
  • ③ 픽셀의 공간적 위치 정보를 함께 포함한다.
  • ④ 영상의 통계함수에 해당한다.
  • ⑤ 관심영역 내 매트릭스를 관찰하여 각 농도값에 대한 픽셀 수 통계를 구한다.
✓ 정답 · ③ 픽셀의 공간적 위치 정보를 함께 포함한다. 히스토그램은 위치 정보를 포함하지 않는다. 동일한 픽셀 값 분포를 가진 서로 다른 두 영상은 히스토그램이 같아도 실제 구조는 전혀 다를 수 있다. 이를 히스토그램의 위치 비의존성이라 한다.
